# serve.py → Proto-Native Rewrite Plan

**Requested by:** koole__ (2026-03-25)
**Status:** Approved by framers, not yet started

## Context

serve.py is 1824 lines of handrolled stdlib HTTP (ThreadingTCPServer). It handles routing, parsing, serialization, auth wiring, SSE streaming, OAuth, static files, LLM dispatch, persona CRUD, congress CRUD, task listing, and wallet balance — all in one synchronous file. Features and bugs are hard to fix because of the tangled scope.

## Decision

Rewrite serve.py using:
- **connect-python** (connectrpc.com) for the proto-native API layer
- **buf** for code generation from `.proto` files
- **uvicorn** as the ASGI server (replaces ThreadingTCPServer)
- Python throughout (no language change)

## What Changes

| Layer | Current | After |
|---|---|---|
| Routing | Handrolled do_GET/do_POST/do_PATCH/do_DELETE | Generated by buf from .proto |
| Request parsing | Manual json.loads + field extraction | Generated typed classes |
| Response serialization | Manual json.dumps + send_response | Generated |
| Auth wiring | Manual at each call site (~80 lines) | Middleware |
| SSE streaming | Custom polling loop + _active_streams dict | Server streaming RPC |
| Server | ThreadingTCPServer | uvicorn (async) |
| Temporal activities | Raw aiohttp + manual JSON dicts | Generated typed Python clients |

## What Stays

- All actual logic (LLM calls, DB reads, persona file mutations, Discord inject)
- GitHub OAuth flow (raw HTTP — browser redirect, not RPC)
- GitHub webhook receiver (raw HTTP + HMAC)
- Static file serving (uvicorn middleware)

## 5 Proto Services

```
CongressService
  - StartSession(topic, discord_user?) → (session_id, session_number)
  - PostDebate(task, identity, session_id?) → (response, identity)
  - StreamDebate(session_id) → stream of (identity, text, done)     [server streaming, replaces SSE]
  - ListIdentities() → stream of identity objects
  - ListSessions() → stream of session objects
  - GetSession(session_id) → session
  - PatchSession(session_id, verdict?, status?, evolution?, thread_id?, task_titles?) → ok

PersonaService  [localhost-auth only]
  - ListPersonas() → stream of persona objects
  - GetPersona(name) → persona + prompt
  - CreatePersona(name, display_name, model, ...) → persona
  - UpdatePersona(name, fields...) → persona
  - DeletePersona(name) → ok
  - PostVerdict(name, verdict, date?) → ok

AgentService  [public read]
  - ListAgents() → (active[], fired[], moderator)

TaskService
  - ListTasks() → stream of task objects

WalletService
  - GetBalance() → (address, balance_eth, chain)
```

## Migration Steps

1. Write 5 proto files in `hello-world/proto/` (~100 lines total)
2. Add `buf.gen.yaml` and run `buf generate` → get base service classes + typed clients
3. Lift logic out of serve.py into service class methods (mechanical, method by method)
4. Add uvicorn entry point, wire OAuth + webhook as raw routes
5. Update Temporal activities to use generated Python clients (replace aiohttp calls)
6. Delete serve.py

## Key Notes

- connect-python is pre-1.0 beta (breaking changes expected at 1.0) — acceptable given we're rebuilding anyway
- gRPC-Web protocol NOT supported in connect-python; browser clients use Connect protocol via `@connectrpc/connect-web`
- SSE alternative: server streaming RPC works over HTTP/1.1 (no HTTP/2 required for server streaming)
- Full-duplex bidirectional streaming requires HTTP/2 end-to-end (not needed here)
- Temporal activities stay Python — no change to temporal-workflows repo
